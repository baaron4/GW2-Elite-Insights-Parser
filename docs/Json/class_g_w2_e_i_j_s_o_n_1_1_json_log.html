bool IsInstanceLog
 Indicates that the log represents a full instance log.
In such logs, it is possible for the same player to be present multiple times, that means they either changed specs or subgroups during the instance.
Make sure to only include data for a given player only on phases that intersect their aware times.
Individual encounter states (Success, CM, etc) can be checked on phases with JsonPhase.PhaseType set to "Encounter".

IReadOnlyList< IReadOnlyList< long > >? PresentInstanceBuffs
 List of present instance buffs, values are arrays of 3 elements, value[0] is buff id, value[1] is number of stacks, value[2] the index in JsonLog.Phases where the buffs are relevant.
value[2] is mainly relevant in instance logs, it can either point towards a specific Encounter phase for encounter specific buffs or to the Instance phase for buffs covering the whole instance, for example fractal instabilities.
In boss logs, value[2] will always be the "Full Fight" phase.
